How to Manage Posts and the Slideshow

Adding posts

You will notice that there are several posts on the front page (below the slideshow box). Automatically the 4 most recent posts will be featured on the front page. If you want to add a post to announce something, you should do it this way to ensure the post will be on the front page. If you want to keep a post on the front page for a long time you should use the slideshow feature instead.

Post header:

Posts have a slightly different header as can be seen below.

---
title: 2018 Congress Documents and Journal Articles are Now Available
layout: post
folder_str: ./../../../
picture: ./pictures/RERCI-release-img.png
excerpt_separator: <!--stop-->
---

The title section will become the title of the post. The picture section should be a link to a picture on the web server. This picture will be used on the fron page. You should not change any of the other fields.

The front page also contains an excerpt of the post. All text from the start of the article until the <!--stop--> will be used as the excerpt. Hence you will need to put <!--stop--> after the section of text you want to be excerpt. An example is given below.

Welcome to the new SERCI website! Take a few minutes to look around and familiarise yourself with the new look. No files, documents or articles should have been removed in the transition from the old website to this one. If you're struggling to find something feel free to email [webmaster@serci.org](mailto:webmaster@serci.org) and we'll help you from there. We've also made a [SERCI twitter account](https://twitter.com/SerciCopyright) so please follow us if you want to stay up to date.
{:.body-text}
<!--stop-->

Post title:

The filename for posts is very important. The date must be the first thing in the filename followed by the actual filename. You should only use "-" as a seperator. An example of a filename is given below.

2018-17-10-this-will-be-the-post-filename.md

Jekyll will read this filename and create an html file called "this-will-be-the-post-filename.html".

Changing the slideshow

You can control the slideshow on the front page by changing the slideshow_data.txt file. Each line of that document reprisents a slide in the slideshow. There should always be exactly 3 slides, hence, there should always be exactly 3 lines in the file. If you insert and extra blank line anywhere in the file (even at the end) this will be interpreted as 4th slide and it will break the slide show.

Each line has 3 fields, the slide title, a link to the slide picture and link to where users will be taken to when they click on the slide. These 3 fields are seperated by the "~" character. There must always be these 3 fields and they must always be in the order mentioned. An example is shown below. Note that there are no spaces between the fields, only the "~" character.

SERCIAC 2019 in Montpellier~./pictures/article_pictures/website_css.jpg~./annual_congress.html
